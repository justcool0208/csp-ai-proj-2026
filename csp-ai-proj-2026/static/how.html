CSP SHEMS

System Logic: Constraint Satisfaction Problem

Mathematical modeling of energy scheduling using the Google CP-SAT solver.

Model Formulation

1. Temporal Domain

The 24-hour cycle is discretized into 96 time-steps (15 minutes each). Decision variables are mapped to these integer steps.

2. Discrete Interval Variables

Appliances are modeled as interval variables with specific durations, ensuring they cannot be interrupted once started.

3. Hard System Constraints

Global power limits, battery state-of-charge (SOC) balance, and mandatory appliance windows are strictly enforced.

Multi-Objective Optimization

The solver minimizes a weighted objective function to prioritize cost reduction and grid stability:

Minimize: (TotalCost * 1,000,000) + (PeakGridLoad * 100) + Sum(WeightedStartTimes)

This formulation ensures that the system first seeks the absolute lowest tariff cost, then focuses on lowering peak demand, and finally optimizes for user priority.

CP-SAT Engine

Unlike heuristic methods, the Google OR-Tools CP-SAT solver uses advanced SAT-based search techniques to prove optimality. This ensures that the generated schedule is mathematically the most efficient possible within the provided constraints.
